Thought for 2 seconds## Explicación del script build-ts-docker.sh

El archivo `build-ts-docker.sh` es un script de shell (bash) que he creado para simplificar y automatizar el proceso de construcción de tu imagen Docker. Vamos a explicar para qué sirve y cómo usarlo:

### ¿Para qué sirve?

El script `build-ts-docker.sh` tiene varios propósitos:

1. **Automatización**: Ejecuta el comando de construcción de Docker con los parámetros correctos.
2. **Feedback mejorado**: Proporciona mensajes claros sobre el éxito o fracaso del proceso.
3. **Instrucciones**: Muestra automáticamente el siguiente paso (cómo ejecutar el contenedor) si la construcción es exitosa.
4. **Consistencia**: Asegura que siempre uses los mismos parámetros al construir la imagen.
5. **Documentación**: Sirve como referencia para recordar cómo construir la imagen correctamente.


### Contenido del script

```shellscript
#!/bin/bash

echo "Construyendo imagen Docker para BarkAndMeow..."
docker build -t topgambajrjdeveloper/barkandmeow .

if [ $? -eq 0 ]; then
  echo "✅ Imagen construida correctamente: topgambajrjdeveloper/barkandmeow"
  echo "Para ejecutar el contenedor:"
  echo "docker run -p 3000:3000 --env-file .env topgambajrjdeveloper/barkandmeow"
else
  echo "❌ Error al construir la imagen Docker"
fi
```

### Cómo usar el script

Para usar el script, sigue estos pasos:

1. **Dar permisos de ejecución** (solo necesitas hacerlo una vez):

```shellscript
chmod +x build-ts-docker.sh
```


2. **Ejecutar el script**:

```shellscript
./build-ts-docker.sh
```

·. **Publicar desde Docker hacia Render**

```
docker push topgambajrjdeveloper/barkandmeow
```




Esto ejecutará el comando `docker build -t topgambajrjdeveloper/barkandmeow .` y mostrará mensajes informativos sobre el resultado.

### Ventajas de usar este script

1. **Menos escritura**: No necesitas recordar y escribir el comando completo cada vez.
2. **Menos errores**: Evita errores tipográficos al escribir comandos largos.
3. **Feedback claro**: Muestra mensajes más amigables que los comandos de Docker por defecto.
4. **Documentación integrada**: El script muestra el siguiente paso a seguir.
5. **Extensibilidad**: Puedes ampliar el script para añadir más funcionalidades, como:

1. Verificar si Docker está en ejecución
2. Limpiar imágenes antiguas
3. Ejecutar automáticamente el contenedor después de construirlo
4. Añadir opciones para diferentes entornos (desarrollo, producción)

Thought for 1 second### Guía para configurar PayPal y recibir donaciones

Para recibir donaciones a través de PayPal en tu aplicación BarkAndMeow, necesitas configurar una cuenta de desarrollador de PayPal y obtener las credenciales necesarias. Aquí te guío paso a paso:

## 1. Crear una cuenta de PayPal Business

Si aún no tienes una cuenta de PayPal Business:

1. Ve a [PayPal.com](https://www.paypal.com)
2. Haz clic en "Registrarse" y selecciona "Cuenta de empresa"
3. Sigue los pasos para completar el registro
4. Verifica tu cuenta siguiendo las instrucciones que recibirás por correo electrónico


## 2. Acceder al Portal de Desarrolladores de PayPal

1. Ve a [PayPal Developer](https://developer.paypal.com)
2. Inicia sesión con tu cuenta de PayPal Business
3. Una vez dentro, verás el Dashboard de desarrollador


## 3. Crear una aplicación para obtener credenciales

1. En el Dashboard de desarrollador, haz clic en "Apps & Credentials" en el menú superior
2. Haz clic en "Create App" bajo "REST API apps"
3. Proporciona un nombre para tu aplicación (por ejemplo, "BarkAndMeow Donations")
4. Selecciona el tipo de cuenta que deseas utilizar:

1. **Sandbox**: Para pruebas (recomendado para empezar)
2. **Live**: Para producción (cuando estés listo para recibir donaciones reales)



5. Haz clic en "Create App"


## 4. Obtener las credenciales

Después de crear la aplicación, serás redirigido a la página de detalles de la aplicación donde encontrarás:

1. **Client ID**: Es una cadena larga que identifica tu aplicación
2. **Secret**: Es tu clave secreta (haz clic en "Show" para verla)


Estas son las credenciales que necesitas para configurar la integración de PayPal en tu aplicación.

## 5. Configurar las variables de entorno

Ahora, necesitas configurar estas credenciales como variables de entorno en tu aplicación:

1. Para el entorno de desarrollo (Sandbox):

```plaintext
NEXT_PUBLIC_PAYPAL_CLIENT_ID=tu_client_id_de_sandbox
PAYPAL_SECRET=tu_secret_de_sandbox
```


2. Para el entorno de producción (Live):

```plaintext
NEXT_PUBLIC_PAYPAL_CLIENT_ID=tu_client_id_de_produccion
PAYPAL_SECRET=tu_secret_de_produccion
```




## 6. Configurar los permisos de la aplicación

1. En la página de detalles de tu aplicación, asegúrate de que los siguientes permisos estén habilitados:

1. **Transaction Search**: Para buscar y verificar transacciones
2. **Payments**: Para procesar pagos
3. **Tracking**: Para seguimiento de pagos



2. Haz clic en "Save" si realizas cambios en los permisos


## 7. Configurar Webhooks (opcional pero recomendado)

Los webhooks te permiten recibir notificaciones automáticas cuando ocurren eventos como pagos completados:

1. En la página de detalles de tu aplicación, desplázate hacia abajo hasta la sección "Webhooks"
2. Haz clic en "Add Webhook"
3. Ingresa la URL de tu webhook (por ejemplo, `https://tudominio.com/api/webhooks/paypal`)
4. Selecciona los eventos que deseas recibir (al menos "Payment capture completed")
5. Haz clic en "Save"


## 8. Probar la integración

Antes de pasar a producción, es importante probar la integración en el entorno Sandbox:

1. Usa las credenciales de Sandbox en tu aplicación
2. Realiza donaciones de prueba utilizando cuentas de Sandbox
3. Verifica que los webhooks funcionen correctamente
4. Comprueba que las donaciones se registren en tu base de datos


## 9. Pasar a producción

Cuando estés listo para recibir donaciones reales:

1. Cambia las credenciales de Sandbox por las de Live en tus variables de entorno
2. Actualiza la URL del webhook si es necesario
3. Realiza una pequeña donación de prueba para verificar que todo funciona correctamente


## 10. Personalizar la experiencia de donación

Puedes personalizar la experiencia de donación en PayPal:

1. En el Dashboard de PayPal Business, ve a "Configuración de la cuenta"
2. Personaliza el logotipo, colores y otros elementos visuales
3. Configura mensajes de agradecimiento personalizados para los donantes


## Consideraciones adicionales

- **Comisiones**: PayPal cobra comisiones por las donaciones. Asegúrate de entender la estructura de tarifas.
- **Impuestos**: Consulta con un asesor fiscal sobre las implicaciones fiscales de recibir donaciones.
- **Cumplimiento normativo**: Dependiendo de tu país, puede haber requisitos legales adicionales para recibir donaciones.
